10주차 <모바일 컴퓨팅> 모바일 통신, 인터넷
WWW. WEB, 웹은 하이퍼텍스트 연결됨. 서버-클라이언트 개념이있음.

모바일플랫폼은 안드로이드처럼 OS(운영체제)를 말함.

다중접속방식
★★★★★
FDMA. Frequency. 주파수분할. 제일 안좋음.
TDMA. TIME. 시분할. FDMA 8배.
CDMA. CODE 분할. 패킷처럼 같은 코드로 전송. FDMA 20배

 * FDMA: Frequency Division Multiple Access. 주파수 분할 다중 접속. 주파수에 파장을 쪼개서 보내주는거임. 손실이 발생.
 * TDMA: Time Division Multiple Access. 시분할 다중 접속. 시간을 쪼개서 보내줌. FDMA보다 8배 효율. 
 * CDMA: Code Division Multiple Access. 코드분할 다중 접속. 특정 코드로만 보내줌. FDMA보다 20배 효율. (이거 주로 채택)
 * GMA라는 것도 있더라. 이건 따로 공부해보셈.
(GSM 유럽꺼.)

13주차: <클라우딩 컴퓨팅> 클라우드 컴퓨팅의 개요, 특징

클라우드 컴퓨팅: 개인PC가 아니라 구름, 즉 가상환경에 저장되있는 거.
장: 자원절약. 인력절감.
단: 보안성. 장애(서버다운되면 리스크), 호환성.

★★★★★
클라우드 유형 SaaS. PaaS. IaaS.
IaaS: 모든 인프라를 때려넣음. 스토리지, 서버, 네트워크. (아마존서비스를 말함.)
PaaS: 개발환경만 제공. 구글앱. sw환경. 구축해놓은 환경만 제공.(구글 앱)
SaaS: 인터넷 통해 SW만 사용하는 서비스.

	 * IaaS: 모든 인프라를 담은 클라우드 서비스. (Infrastructure as-a Service)
	 * PaaS: 개발환경을 담은 '플랫폼'을 담은 클라우드 서비스 (Platform)
	 * SaaS: '소프트웨어'만을 담은 서비스. (Software)
	 
클라우드 컴퓨팅 이슈. "보안, 성능 딸린다."

14주차: <정보 보안> 정보 보안의 개요

정보보안? 모든 정보보호 활동.
기밀성. 비밀성. 무결성.

웜,바이러스,트로이목마. 이런 게 악의적 바이러스.

이걸 방지하려고 '암호화기술' 2가지 있음.


암호(해독 불가능하게. or 해독가능하게)-암호화.
평문(의미 ㅇ)-암호문(암호화).
암호화? 알고리즘 + 키. 암호화+복호화.

평문(M)>암호 알고리즘(암호화E) > 암호문(c) > 암호알고리즘(복호화D)>복호문(평문M)
여기에서 암호 알고리즘 (E 함호화할 때 필요한 암호키 K1, D복호화할 때 필요한 암호키 K2)

K1=K2 : 비밀키 (누출되면 털리니까)
K1≠K2: 공개키 (누출되도 됨)

규칙은 2가지. 대칭키(비밀키). 비대칭키(비대칭키)
문을 닫을때, 열때
같은거를 비밀키. 열때만 필요한거는 비대칭키.

★★★★★비밀키 암호화, 공개키 암호화.

"비밀키"
송신-수신자 같은 키 가지고 있을때.
특) 키 보안에 의존. 복잡할수록 열기 어렵다.
단) 1 N명이 접근할 때 N개의 비밀키 필요. 2 누가 썼는지 로그안남음 (부인방지 라고함)

"공개키"
특) 1 디퍼-힐만이 제안. 두개의 분리된 키. 2 공개키1개, 비공개키1개 한쌍씩 생겨남.
어디에 사용? 전자서명. 복호화든, 첨에 안호화를 하든 하나만 알려주는 거.
단) 누구든 복호화(OR 암호화) 가능. 그래서 기밀성 보장하려면 디지털 서명보장하던지 수신자는 공개키로 다시 암호화하던지.
장) 편의개선. 부인방지+봉쇄
단) 위장공격에 취약.

웹보안. 1994 S-Http

SSL. 통신망 스택에서 HTTP보다 하위에 작용해 비밀적용. 데이터 무결성 제공.
